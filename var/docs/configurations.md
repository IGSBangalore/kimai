    <article class="markdown-body entry-content" itemprop="text"><h1><a id="user-content-configurations" class="anchor" aria-hidden="true" href="#configurations"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Configurations</h1>
<h2><a id="user-content-environment-specific-settings-env" class="anchor" aria-hidden="true" href="#environment-specific-settings-env"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Environment specific settings (.env)</h2>
<p>The most basic settings, which need always be adjusted are stored in the <code>.env</code> file:</p>
<ul>
<li><code>MAILER_URL</code> - smtp connection for emails</li>
<li><code>MAILER_FROM</code> - application specific "from" address for all emails</li>
<li><code>APP_ENV</code> - environment for the runtime (use <code>prod</code> if you are unsure)</li>
<li><code>DATABASE_URL</code> - database connection for storing all application data</li>
<li><code>DATABASE_PREFIX</code> - precix for any Kimai table in the configured database</li>
<li><code>APP_SECRET</code> - secret used for hasing user password (if you cahnge this, every password is invalid afterwards)</li>
</ul>
<h2><a id="user-content-config-files" class="anchor" aria-hidden="true" href="#config-files"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Config files</h2>
<p>Configuration of Kimai is spread in all files in the <code>config/</code>directory but mainly it these files:</p>
<ul>
<li><code>.env</code> - environment specific settings</li>
<li><code>config/packages/kimai.yaml</code> - Kimai specific settings</li>
<li><code>config/packages/admin_lte.yaml</code> - Kimai base theme</li>
<li><code>config/packages/fos_user.yaml</code> - user management and email settings</li>
<li><code>config/packages/local.yaml</code> - your local configuration settings</li>
</ul>
<p>There are several other configurations that could potentially be interesting for you in <a href="/kevinpapst/kimai2/blob/master/config/packages">config/packages/*.yaml</a>.</p>
<p>If you want to adjust a setting from any of these files, use <code>local.yaml</code> (see below).</p>
<h4><a id="user-content-other-topics" class="anchor" aria-hidden="true" href="#other-topics"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Other topics</h4>
<ul>
<li><a href="/kevinpapst/kimai2/blob/master/var/docs/theme.md">Theme settings</a> - in <code>kimai.yaml</code> and <code>admin_lte.yaml</code></li>
<li><a href="/kevinpapst/kimai2/blob/master/var/docs/emails.md">Email configuration</a> - in <code>swiftmailer.yaml</code></li>
<li><a href="/kevinpapst/kimai2/blob/master/var/docs/dashboard.md">Dashboard widgets</a> - in <code>kimai.yaml</code></li>
<li><a href="/kevinpapst/kimai2/blob/master/var/docs/calendar.md">Calendar</a> - in <code>kimai.yaml</code></li>
</ul>
<h2><a id="user-content-overwriting-local-configs-localyaml" class="anchor" aria-hidden="true" href="#overwriting-local-configs-localyaml"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Overwriting local configs (local.yaml)</h2>
<p>You can create the file <code>config/packages/local.yaml</code> and store your own settings inside. This file will NEVER be shipped with Kimai.
So having your custom settings in this file allows you to easily update Kimai. This is the same concept which is used for the <code>.env</code> file.</p>
<p>An example <code>config/packages/local.yaml</code> file might look like this:</p>
<div class="highlight highlight-source-yaml"><pre><span class="pl-ent">kimai</span>:

<h2><a id="user-content-timesheets-kimaiyaml" class="anchor" aria-hidden="true" href="#timesheets-kimaiyaml"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Timesheets (kimai.yaml)</h2>
<h3><a id="user-content-descriptions-with-markdown" class="anchor" aria-hidden="true" href="#descriptions-with-markdown"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Descriptions with Markdown</h3>
<p>The description for every timesheet entry can be formatted in two different ways, configured with the <code>markdown_content</code> setting.</p>
<ul>
<li><code>false</code> - simple newlines in the description box will be displayed in the frontend as well (default)</li>
<li><code>true</code> - description will be rendered with a markdown engine, supporting simple lists and other HTML content</li>
</ul>
<p>Allowing Markdown in timesheet descriptions is beautiful, but also could be a <a href="https://github.com/erusev/parsedown/blob/master/README.md#security">security risk</a>.
Kimai will only apply the markdown in the user timesheet and not in the admin section as additional security measure.</p>
<h3><a id="user-content-duration-only" class="anchor" aria-hidden="true" href="#duration-only"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Duration only</h3>
<p>Kimai supports two modes for displaying and recording timesheet entries:</p>
<ul>
<li><code>begin</code> and <code>end</code> time (default)</li>
<li><code>date</code> and <code>duration</code> (the so called <code>duration_only</code> mode)</li>
</ul>
<p>When activating the <code>duration_only</code> mode all timesheet tables will only display the <code>date</code> and <code>duration</code> of all records.
In addition, the "edit timesheet" forms will be changed and instead of displaying the <code>end</code> date you will see a field for <code>duration</code>.
The <code>start</code> date is only visible in these forms when editing an active or starting a new record.</p>
<p>You can activate the <code>duration_only</code> mode by switching the configuration key <code>kimai.timesheet.duration_only</code> to <code>true</code> in the file <a href="/kevinpapst/kimai2/blob/master/config/packages/kimai.yaml">kimai.yaml</a>.</p>
<p>For supported formats while entering the <code>duration</code> please see the <a href="/kevinpapst/kimai2/blob/master/var/docs/timesheet.md">timesheet chapter</a></p>
<h3><a id="user-content-rounding-of-begin-end-and-duration-for-timesheet-records" class="anchor" aria-hidden="true" href="#rounding-of-begin-end-and-duration-for-timesheet-records"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Rounding of begin, end and duration for timesheet records</h3>
<p>Rounding rules are used to round the begin &amp; end dates and the duration for timesheet records.</p>
<ol>
<li>You can define as many rules as you want ("default" is only an example)</li>
<li>Every matching rule will be applied, so be careful with overlapping rules</li>
<li>The end_date of timesheet records will be used to match the day (think about entries which are recorded overnight)</li>
<li>If you set one of "begin", "end", "duration" to 0 no rounding will be applied for that field and the exact time (including seconds) is used for calculation</li>
<li>The values of the rules are minutes (not the minute of an hour), so 5 for "begin" means we round down to the previous multiple of five</li>
<li>You can define different rules for different days of the week</li>
<li>"begin" will always be rounded to the floor (down) and "end" &amp; "duration" to the ceiling (up)</li>
<li>Rounding rules will be applied on stopped timesheet records only, so you might see an un-rounded value for the start time and duration until you stop the record</li>
</ol>
<p>You can configure your <code>rounding</code> rules by changing the configuration file <a href="/kevinpapst/kimai2/blob/master/config/packages/kimai.yaml">kimai.yaml</a>.</p>
<h4><a id="user-content-examples" class="anchor" aria-hidden="true" href="#examples"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Examples</h4>

<div class="highlight highlight-source-yaml"><pre><span class="pl-ent">kimai</span>:
    <span class="pl-ent">timesheet</span>:

                <span class="pl-ent">days</span>: <span class="pl-s">['saturday','sunday']</span>
                <span class="pl-ent">begin</span>: <span class="pl-c1">1</span>
                <span class="pl-ent">end</span>: <span class="pl-c1">1</span>
                <span class="pl-ent">duration</span>: <span class="pl-c1">60</span></pre></div>
<p>A rule which is often used is to round up to a mulitple of 10:</p>
<div class="highlight highlight-source-yaml"><pre><span class="pl-ent">kimai</span>:
    <span class="pl-ent">timesheet</span>:
        <span class="pl-ent">rounding</span>:
            <span class="pl-ent">workdays</span>:
                <span class="pl-ent">days</span>: <span class="pl-s">['monday','tuesday','wednesday','thursday','friday','saturday','sunday']</span>
                <span class="pl-ent">begin</span>: <span class="pl-c1">10</span>
                <span class="pl-ent">end</span>: <span class="pl-c1">10</span>
                <span class="pl-ent">duration</span>: <span class="pl-c1">0</span></pre></div>
<h3><a id="user-content-hourly-rates-for-timesheet-records" class="anchor" aria-hidden="true" href="#hourly-rates-for-timesheet-records"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Hourly rates for timesheet records</h3>
<p>If you want to apply different hourly rates multiplication <code>factor</code> for specific weekdays, you can use this <code>rates</code> configuration.</p>
<ol>
<li>You can define as many rules as you want ("workdays" and "weekend" are only examples)</li>
<li>Every matching rule will be applied, so be careful with overlapping rules</li>
<li>The end_date of timesheet records will be used to match the day (think about entries which are recorded overnight)</li>
<li>"days" is an array of weekdays, where the days need to be written in english and in lowercase</li>
<li>"factor" will be used as multiplier for the applied hourly rate</li>
<li>Rate rules will be applied on stopped timesheet records only, as it can't be calculated before</li>
<li>There is no default rule active, by default the users hourly-rate is used for calculation</li>
</ol>
<p>You can configure the <code>hourly_rate</code> rules by changing the configuration file <a href="/kevinpapst/kimai2/blob/master/config/packages/kimai.yaml">kimai.yaml</a>.</p>
<h4><a id="user-content-examples-1" class="anchor" aria-hidden="true" href="#examples-1"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Examples</h4>
<ol>
<li>The "workdays" rule will use the default "hourly rate" for each timesheet entry recorded between "monday" to "friday" as a multiplication with 1 will not change the result</li>

</ol>
